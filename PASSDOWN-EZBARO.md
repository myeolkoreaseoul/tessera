# 이지바로 상시점검 자동화 — 패스다운 문서

> **목적**: 이 문서를 AI(Gemini CLI, Claude Code 등)에게 넘겨주면, PC2에서 이지바로 상시점검 자동화를 구축할 수 있도록 한다.
> **작성일**: 2025-02-12
> **작성 배경**: e나라도움 RPA에서 검증된 프로세스를 이지바로에 적용

---

## 1. 검증된 프로세스 아키텍처 (e나라도움에서 증명됨)

```
Phase 0: 브라우저 자동화로 점검 대상 페이지 이동
Phase 1: 그리드에서 집행내역 전체 추출
Phase 2: 각 행의 첨부파일 다운로드
Phase 3: 첨부파일 OCR (PDF, HWP, Excel, 이미지)
Phase 4: data.json 저장
Phase 5: AI가 OCR 텍스트를 직접 읽고 판정 → results.json 작성
Phase 6: 브라우저 자동화로 점검의견 입력 + 보완요청
```

### 핵심 원칙
- Phase 5(판정)는 **AI가 직접 문서를 읽고 종합 판단**한다
- 키워드 매칭 기반 rule-based judge는 70% 오탐률로 폐기됨
- API 종량제 호출도 비용 문제로 폐기됨
- **AI CLI가 대화 중에 data.json을 읽고, 가이드라인을 참조하여, results.json을 직접 작성**하는 것이 정답

## 4. 기술적 노하우 (e나라도움에서 배운 교훈)

### 4.1 브라우저 자동화
- **Chrome 디버그 모드** 필수: `--remote-debugging-port=9444 --user-data-dir=C:\chrome-debug-profile`
- Playwright의 `chromium.connectOverCDP('http://localhost:9444')`로 연결
- 이미 로그인된 세션을 그대로 사용 (자동 로그인 시도 금지)
- 세션 타임아웃 방지: 주기적으로 세션 연장 버튼 클릭 (5분 간격)

### 4.2 그리드 행 선택
- IBSheet의 `selectRow()`는 버그가 있음 → `grid.focus(row)` 사용
- 이지바로의 그리드 라이브러리에 따라 적절한 행 선택 방법 파악 필요
- DOM 클릭은 신뢰성 떨어짐 → 그리드 API 사용 권장

### 4.3 첨부파일 다운로드
- ZIP 형태로 다운로드되는 경우 자동 압축 해제
- 다운로드 폴더 경로를 절대경로로 지정
- 파일명에 한글이 포함될 수 있으므로 인코딩 주의

### 4.4 OCR
- PDF → pdf-parse (텍스트 추출)
- HWP → hwp5txt (한글 파일)
- Excel → exceljs
- 이미지 → tesseract (정확도 낮으므로 보조 수단)
- **텍스트 상한 12,000자** (너무 짧으면 중요 정보 누락)
- OCR 공백 문제: 한국어 "회 의 록" → fuzzy matching 필요

### 4.5 AI 판정 — 핵심 원칙
1. **키워드 매칭 하지 말 것** — "내부결재"라는 글자가 없어도 "품의서"가 내부결재임을 알아야 함
2. **파일명만 보지 말 것** — 파일 내용(OCR 텍스트)을 반드시 읽어야 함
3. **문서 유형을 종합 판단** — 제목, 결재선, 내용, 금액 등을 종합
4. **보수적 판단 원칙** — 확실한 경우만 "적정/점검완료", 나머지 "확인/보완요청"
5. **일시 임차료에 계약서 요구하지 말 것** — 세금계산서 + 지출결의서면 충분
6. **전자세금계산서는 시스템 자동 연동** — 별도 첨부 불요

### 4.6 results.json 형식

```json
  "issues": ["참여연구자현황표 미첨부", "계좌이체증명 미첨부"],

  "comment": "1. 참여연구자현황표(연구자명, 참여기간, 인건비계상률 등) 2. 계좌이체증명 등 보완 바랍니다."

```

### 4.7 페이지 네비게이션
- 상세 페이지 진입 후 목록 복귀 시 필터/설정이 초기화될 수 있음
- 복귀 후 반드시 필터 재설정 + 재검색
- 페이지 크기가 리셋되면 표준 페이지네이션으로 처리 (pagesize=20)


### Step 3: CLAUDE.md 내용 (AI에게 주는 프로젝트 컨텍스트)
```markdown
# 이지바로 상시점검 자동화

## 역할
통합이지바로(ezbaro.kr)에서 국가R&D 과제의 집행내역을 상시점검한다.
집행건별로 첨부 증빙을 OCR로 읽고, 혁신법 매뉴얼 기준으로 판정하여,
점검의견을 입력하고 보완요청까지 자동 수행한다.

## 판정 방식
- data.json의 OCR 텍스트를 직접 읽고 종합 판단
- guidelines/ 폴더의 혁신법 기준 참조
- 키워드 매칭 금지 — 문서의 맥락을 종합적으로 판단할 것
- 보수적 판단: 확실한 경우만 점검완료, 나머지 보완요청

## 실행 순서
1. node run-all.js --skip-judge --skip-review 로 수집
2. 대화에서 data.json 읽고 판정 → results.json 작성
3. node lib/review-ezbaro.js --results=xxx-results.json --save 로 입력

## 핵심 주의사항
- Chrome 디버그 모드 필수 (포트 9444)
- 그리드 행 선택은 API 사용 (DOM 클릭 비추)
- 상세→목록 복귀 시 필터 초기화됨 → 재설정 필수
- OCR 텍스트 12,000자 상한
- "품의서" = 내부결재문서로 인식해야 함
```

### Step 4: 이지바로 DOM 분석 (첫 작업)

