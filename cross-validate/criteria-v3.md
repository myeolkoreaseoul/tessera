# 정산검토 판정 기준 v3 (심층분석 통합)

> v2 기준 + deep-analyze.js 심층분석 결과(analysis 필드)를 활용한 판정 규칙 추가.
> 이 파일은 사업별로 커스터마이징하여 사용 (기관명, 한도 등 치환).

---

## 공통 규칙 (v2 유지)

1. **0원 항목** → SKIP (취소전표)
2. **evidenceSub**에 "소득 지급명세서" 포함 → 시스템 내장 증빙 (파일 없어도 인정)
3. **evidenceType = "보조금전용카드"** → 카드전표로 세금계산서 대체 가능
4. **evidenceType = "전자세금계산서"/"전자계산서"** → 시스템 내장 결제증빙
5. **files=0 + evidenceSub=""** → 증빙 없음 → **확인**
6. **보수적 원칙**: 확실한 경우만 "적정", 불확실하면 "확인"
7. **"기타(첨부파일필수확인)"** → 별도 첨부파일 필수

## 오판 방지 규칙 (v2 유지)

- OP-1: "집행취소 협조 요청" PDF = 참고문서, 취소건 아님
- OP-2: 출장복명서 = 사업별로 필수 여부 다름 (analysis.fields 참조)
- OP-3: vendorName = 기관명은 정상 (자체 집행)
- OP-4: 자문료 vendorName은 기관/개인 모두 정상
- OP-5: 위촉직 급여 = 지출결의서 + 급여내역 + 위촉직급여정보 → 적정
- OP-6: 인쇄비/수수료 = 견적서2개 + 구매절차 + 거래명세서 → 적정
- OP-7: 회의비 = 방명록 존재 → 인원 확인 가능
- OP-8: 동일 데이터 → 동일 판정

## ★ 심층분석 규칙 (v3 신규 — analysis 필드 기반)

> deep-analyze.js가 각 행에 `analysis: { flags: [...], fields: {...} }` 추가.
> 아래 규칙은 analysis.flags를 참조하여 판정에 반영.

### DR-1: 원천세 납부증빙

- **플래그**: `원천세_납부증빙_미첨부`
- **조건**: analysis.fields.type = "원천세" + analysis.fields.taxPaymentProof = false
- **판정**: → **확인**
- **보완요청**: "원천세 집행건에 대해 국세청 납부 여부를 확인할 수 있는 증빙(원천징수이행상황신고서 등)이 첨부되어 있지 않습니다. 해당 원천세가 실제 납부되었음을 확인할 수 있는 서류를 제출하여 주시기 바랍니다."
- **근거**: 원천세는 월별/반기별 일괄 신고·납부. 공제만 하고 미납 가능성 있음.

### DR-2: 내부인건비 흡수

- **플래그**: `내부인건비_산출근거_미비`
- **조건**: analysis.fields.type = "내부인건비흡수" + analysis.fields.missingDocs.length > 0
- **판정**: → **확인**
- **보완요청**: "내부인건비 흡수 집행건에 대하여 인건비 산정기준(실제월지급인건비 × 참여개월 × 참여율)의 적정성을 확인할 수 있도록 다음 서류를 제출하여 주시기 바랍니다. 1) 사업 참여인력 명단 및 개인별 참여율 2) 참여인력의 급여대장 3) 기관부담 4대보험료 산출내역"
- **근거**: 구조는 합법이나 산출근거 없으면 정정성 확인 불가.

### DR-3: 자문료 비목분류

- **플래그**: `자문료_교통비합산_영수증있음`
- **조건**: analysis.fields.estimatedTransport > 0 + 교통비영수증 있음
- **판정**: → **적정** (단, 비목분류 보완요청)
- **보완요청**: "교통비영수증(KTX) 및 지출발의서상 자문수당과 교통비가 합산 집행된 것을 확인하였습니다. 다만, 외부전문가 교통비는 여비 비목에서 별도 집행하는 것이 원칙이므로, 전문가활용비에 교통비를 합산 집행한 사유를 확인하여 주시기 바랍니다."

- **플래그**: `자문료_한도초과_교통비영수증_없음`
- **조건**: 세전 > 한도 + 교통비영수증 없음
- **판정**: → **확인**

### DR-4: 출장복명서

- **플래그**: `출장복명서_미첨부`
- **조건**: 여비 건 + 출장복명서(결과보고) 없음
- **판정**: 사업별 세칙에 따라 다름
  - 출장복명서가 필수인 사업 → **확인**
  - 출장복명서가 선택인 사업 → 적정 (OP-2 적용)
- **주의**: 출장신청서 ≠ 출장복명서. 통합 서식일 경우 OCR에서 "복명" 키워드 확인.

### DR-5: 재원변경 중복

- **플래그**: `재원변경_중복건`
- **조건**: analysis.fields.duplicateOf 존재
- **판정**: → **SKIP** (원본건만 검토)
- **근거**: 원래 집행 취소 → 재원 변경 → 재집행. 이중 계상 아님.

### DR-6: 공용첨부파일 미수집

- **플래그**: `공용첨부파일_미수집`
- **조건**: files=0 + 공용첨부 ID 존재
- **판정**: → **PENDING** (수동 확인 필요)

### DR-7: 부가세 포함 집행

- **플래그**: `부가세_포함_집행`
- **조건**: vat > 0 + totalAmount = supplyAmount + vat
- **판정**: → **확인** (영리법인 매입세액공제 가능분)
- **보완요청**: "부가가치세가 집행액에 포함되어 있습니다. 환급 가능한 매입세액이 포함된 경우 불인정 대상이므로 확인하여 주시기 바랍니다."

### DR-8: 2천만원 초과 경쟁입찰

- **플래그**: `2천만원초과_경쟁입찰_확인필요`
- **조건**: 물품/용역 공급가액 > 2천만원
- **판정**: → **확인** (경쟁입찰 절차 확인 필요)

### DR-9: 증빙 완전 부재

- **플래그**: `증빙_완전부재`
- **조건**: files=0 + 시스템 내장 증빙 없음 + evidenceSub 없음
- **판정**: → **확인**

## 비목별 판정 기준 (v2 유지, 아래 사업별로 커스터마이징)

### 여비
- 필수: 출장신청서 + 지출결의서 + 교통비 영수증
- 관내출장 (20,000원 수준): 신청서 + 결의서 충분
- analysis.fields.hasReturnReport 참조 (DR-4)

### 인건비
- 위촉직 급여: OP-5 적용
- 내부인건비 흡수: DR-2 적용
- 퇴직연금: 결의서 + 납부서 → 적정

### 자문료/원천세
- 필수: 결의서 + 자문확인서
- 원천세: DR-1 적용
- 교통비 합산: DR-3 적용

### 회의비
- 필수: 결의서 + 회의록/방명록
- 방명록: OP-7 적용

### 사무용품/임차/용역/수수료/재료비/공공요금
- v2 기준 유지

### 대상자사례비
- evidenceSub "소득 지급명세서" → 적정
