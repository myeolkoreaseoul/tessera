# 정산검토 공통 기준 (최상위)

> 이 문서는 **모든 정산검토**(보조금/R&D혁신법 무관)에 공통 적용되는 실질적 검토 기준이다.
> 법체계별 세부 규정은 하위 디렉토리 참조:
> - `보조금/common.md` — 보조금법, 통합관리지침 등
> - `혁신법/common.md` — 국가연구개발혁신법 등 (작성 예정)
>
> 사업별 세칙/지침의 구체적 한도·비율은 해당 사업 guideline 참조.

## 1. 실질적 심층 검토 기준 (자동 검토 시 필수 적용)

> 표면적 체크리스트(증빙 유무, 금액 범위)를 넘어 **회계사 수준의 실질적 검토**를 자동화하기 위한 기준.
> 대구경북재단 180건 수동 검토(2026.02)에서 도출.

### 1.1 원천세 집행건

원천세가 **별도 집행건**으로 시스템에 등록된 경우 (e나라도움, 이지바로, RCMS 등 공통):

**검토 포인트:**
- 원천세 건의 첨부파일이 본건(자문료 등)과 동일한 서류뿐인지 확인
- **납부 증빙이 없으면 보완요청 대상** — 공제만 하고 미납 가능성
- 원천세는 월별/반기별 일괄 신고·납부하므로, 건별 영수증은 없음
- 요청할 증빙: **원천징수이행상황신고서** 또는 **홈택스 납부확인서**

**판정:**
- 납부 증빙 미첨부 → `확인` (보완요청: 납부 증빙 제출 요구)

**보완요청 문구 예시:**
> "원천세 집행건에 대해 국세청 납부 여부를 확인할 수 있는 증빙(원천징수이행상황신고서 등)이 첨부되어 있지 않습니다. 해당 원천세가 실제 납부되었음을 확인할 수 있는 서류를 제출하여 주시기 바랍니다."

### 1.2 내부인건비 흡수 (재단/출연연 등)

기관이 자기 자신에게 인건비를 지급하는 형태 — 일반회계에서 직원 급여 지급 후, 사업비에서 참여율만큼 환입하는 구조.

**구조 자체는 합법.** 정부출연연/재단에서 일반적으로 사용. 하지만 **산출근거 증빙이 반드시 필요.**

**식별 방법:**
- vendorName = 기관 자기 자신
- purpose에 "내부인건비", "흡수", "인건비 배분" 등 키워드
- 비목 = 인건비 > 보수
- 월별 반복 패턴 (1월~12월)

**필수 증빙 체크리스트:**
1. 사업 참여인력 명단 및 개인별 참여율
2. 참여인력의 급여대장 또는 급여 지급 증빙(개인별 계좌이체 내역)
3. 기관부담 4대보험료 산출내역
4. 인건비 산정 공식 확인: `실제월지급인건비 × 참여개월 × 참여율`

**추가 확인:**
- 총사업비 대비 인건비 비율 (사업별 상한 확인 — 보통 30%)
- 동일 인력이 타 사업에도 높은 참여율로 등록되어 있지 않은지 (중복 계상)

**판정:**
- 지출결의서 + 흡수문서만 있고 위 증빙 없음 → `확인` (보완요청)

**보완요청 문구 예시:**
> "내부인건비 흡수 집행건에 대하여, 현재 지출결의서 및 내부인건비 흡수 문서만 첨부되어 있습니다. 인건비 산정기준(실제월지급인건비 × 참여개월 × 참여율)의 적정성을 확인할 수 있도록 다음 서류를 제출하여 주시기 바랍니다. 1) 사업 참여인력 명단 및 개인별 참여율 2) 참여인력의 급여대장 또는 급여 지급 증빙 3) 기관부담 4대보험료 산출내역"

### 1.3 자문료 — 교통비 합산 역산

자문료 실지급액이 한도를 초과하는 것처럼 보일 때, **교통비가 합산**되어 있을 수 있음.

**역산 방법:**
1. 기타소득 원천세율 = **8.8%** (소득세 6.6% + 주민세 2.2% ≒ 8.8%)
2. 실지급액 ÷ (1 - 0.088) = 세전 총액
3. 세전 총액 - 자문료 한도(사업별) = 교통비 추정액
4. 교통비 영수증과 대조

**예시 (1일 한도 60만원 사업):**
- 실지급액 626,550원 → 원천세 역산: 626,550 ÷ 0.912 ≒ 687,000원
- 687,000 - 600,000 = 87,000원 (교통비)
- 교통비영수증 확인: KTX 동대구↔서울 → 근사

**검토 포인트:**
- 지출발의서의 산출근거에 "자문료 ○○원 + 교통비 ○○원" 명시 여부
- 교통비영수증 첨부 여부
- 교통비를 전문가활용비에 합산 vs 여비에서 별도 집행 → 사업별 세칙에 따라 비목분류 확인 필요

**판정:**
- 순수 자문료가 한도 이내 + 교통비 영수증 있음 → `적정` (단, 비목분류 보완요청 가능)
- 역산해도 한도 초과 + 사유서 없음 → `확인`

### 1.4 여비 — 출장복명서 vs 출장신청서

**핵심:** 불인정 기준은 "출장복명서가 없**거나** 출장목적이 명확하지 않은 여비" (OR 조건)

**구분:**
- **출장신청서**: 출장 전 사전 승인 문서 → 복명서가 아님
- **출장복명서(출장보고서)**: 출장 후 결과 보고 → 이것이 필수
- 많은 기관에서 하나의 서식에 "신청+복명"이 같이 있음 → **파일 내용(OCR 텍스트)에서 복명/결과보고 섹션 존재 여부 확인**

**자동 검토 로직:**
1. 파일명에 "복명" 포함 → OK
2. 파일명에 "출장" 포함하되 "신청"만 있음 → OCR 텍스트에서 "복명", "출장결과", "보고" 키워드 검색
3. 위 키워드 없음 → `확인` (출장복명서 미첨부 가능성)
4. 출장목적 필드(purpose)가 명확하면 목적 요건은 충족

### 1.5 재원변경 재집행 (중복건 식별)

동일 지출결의서 번호가 **다른 rowNum에 중복** 등장하는 경우:
- 원래 집행 취소 → 재원(국비↔지방비) 변경 → 재집행
- **이중 계상이 아님** — 하나의 실제 거래

**식별 방법:**
- OCR 텍스트에서 지출결의서 번호 추출 → 중복 검사
- purpose가 동일하고 금액이 동일한 건 → 재원변경 가능성
- "집행취소 협조 요청" 등 파일이 첨부되어 있으면 확실

**판정:**
- 중복건은 하나만 검토, 나머지는 `SKIP` (재원변경 재집행)

### 1.6 공용첨부파일 확인

e나라도움 상세페이지(DD001003S)에는 **개별첨부파일**과 **공용첨부파일** 두 종류가 있음.

- 개별첨부파일: `DD001003S_atchmnflId` — 건별 고유 증빙
- 공용첨부파일: `DD001003S_cmnuseAtchmnflId` — 여러 건이 공유하는 증빙

**검토 포인트:**
- 개별첨부파일 = 0개이지만 공용첨부파일이 존재할 수 있음
- `files: []`이라도 공용첨부에 증빙이 있을 수 있으므로, **공용첨부 미수집 건은 즉시 판정하지 말고 플래그**
- run-all.js에서 `cmnuseAtchmnflId` 필드로 공용첨부 존재 여부 확인 가능

> 이 항목은 e나라도움 특유이지만, 다른 시스템에서도 유사한 공유 첨부 구조가 있을 수 있으므로 최상위에 배치.

## 2. 보완요청 전략

**원칙:** 검토 완료(적정/불인정) 상태에서는 의견란이 없음. **검토 과정에서 확인한 사항을 기관에 전달하려면 보완요청으로 올려야 함.**

**보완요청을 써야 하는 경우:**
- 증빙을 확인했으나 추가 소명이 필요한 경우
- 액션을 보여야 하는 경우 ("이 회계사가 꼼꼼하게 봤다"를 보여줘야 할 때)
- 불인정까지는 아니지만 비목분류, 산출근거 등 확인이 필요한 경우

**보완요청 문구 작성 원칙:**
1. **증빙을 확인했음을 구체적으로 언급** — "지출결의서 및 교통비영수증(KTX)을 확인하였습니다" 등
2. **세칙/지침 조항을 근거로 제시** — "세칙 별표2 여비 참조" 등
3. **요청 사항을 명확히** — "사유를 확인하여 주시기 바랍니다" 등
4. **추측이나 불확실한 표현 금지** — 증빙에 있는 사실만 언급

## 3. 전담기관/운영기관 자체 정산 시 주의사항

검토 대상이 보조금을 관리하는 기관 자체인 경우 (예: 전담기관이 자기 사업비를 정산):
- 해당 기관은 규정을 잘 알고 있으므로, **근거 없는 지적은 역효과**
- 반대로, 규정을 잘 아는 만큼 **증빙 누락은 의도적일 수 있음**
- 세칙/지침의 적용 범위를 먼저 확인 (전담기관 자체에도 적용되는지)
- 보완요청 시 세칙 조항을 정확히 인용하여 전문성 보여야 함
