# e나라도움 시스템 공통

> e나라도움(www.e-narahelp.kr)을 통해 정산검토하는 모든 사업에 공통 적용.
> 상위: `../../common.md` (정산 공통) → `../common.md` (보조금 공통)

---

## 1. 시스템 특성

- IBSheet 기반 그리드 (selectRow 버그 → `grid.focus(row)` 사용)
- 정산구분: 중간정산(002) / 최종정산(009)
- 상세(DD001003S) → 목록(DD001002Q) 복귀 시 정산구분 초기화됨
- 페이지 크기: 상세→복귀 시 20으로 리셋

## 2. 첨부파일 구조

- **개별첨부파일** (`DD001003S_atchmnflId`): 건별 고유 증빙
- **공용첨부파일** (`DD001003S_cmnuseAtchmnflId`): 여러 건이 공유하는 증빙
- `files: []`이라도 공용첨부에 증빙 존재 가능 → 확인 필요

## 3. evidenceSub 해석

- `evidenceSub`에 "소득 지급명세서 외 N개" → 시스템 내장 증빙 존재
- 대상자사례비 + 소득 지급명세서 = 충분 → 적정
- "기타(첨부파일필수확인)" → 별도 첨부파일 필요, files 확인 필수
- `evidenceSub=""` → 증빙 없음

## 4. 전문기관별 하위 디렉토리

```
e나라도움/
├── common.md              ← [이 파일]
└── KHIDI/                 ← 한국보건산업진흥원
    ├── 디지털헬스케어.md
    └── 지역책임의료기관.md
```

> 새 전문기관 추가 시: `e나라도움/{전문기관명}/` 디렉토리 생성
