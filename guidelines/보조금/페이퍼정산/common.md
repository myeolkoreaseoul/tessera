# 페이퍼 정산 공통 검토 기준

## 개요
페이퍼 정산은 e나라도움/보탬e/이지바로 등 웹 시스템이 없는 사업의 정산검토이다.
기관이 **엑셀(집행내역) + PDF(증빙서류)**를 직접 제출하면, 이를 기준에 따라 검토한다.

## 웹 시스템과의 차이점
- **evidenceType 없음**: 전자세금계산서/카드 등 시스템 증빙 유형이 없으므로, PDF 내용으로 증빙 유형을 직접 판단해야 함
- **시스템 자동 증빙 없음**: 웹 시스템에서 자동 확인되는 결제증빙(카드전표, 전자세금계산서)이 없으므로, 모든 건에 대해 증빙서류 확인 필요
- **파일 매칭 수동**: 증빙 PDF가 행별로 폴더 또는 파일명으로 구분되어 제출됨

## 엑셀 제출 형식
기관마다 엑셀 컬럼이 다를 수 있으며, 시스템이 자동감지한다. 최소 필수 컬럼:
- 사용용도/내용
- 금액/집행금액
- 거래처/상호 (권장)
- 비목/세목 (권장)
- 집행일자 (권장)

## PDF 증빙 정리 방법

### 폴더 방식 (권장)
```
evidence/
  ├── 1/    → 1번 항목 증빙
  │   ├── 세금계산서.pdf
  │   └── 지출결의서.pdf
  ├── 2/    → 2번 항목 증빙
  └── ...
```
폴더명은 순번과 일치해야 함 (1, r1, 01 등 허용).

### 파일명 방식
```
evidence/
  ├── 001_세금계산서.pdf      → 1번 항목
  ├── 001_지출결의서.pdf      → 1번 항목
  ├── 002_회의록.pdf          → 2번 항목
```
파일명 접두사가 순번과 일치해야 함.

## 금액 대조 원칙 (CLAUDE.md 공통 원칙 적용)
1. 집행금액을 기준으로, 해당 금액이 증빙서류 PDF 텍스트에 존재하는지 역방향 대조
2. 금액이 존재하면 → 금액 일치 확인
3. 금액이 미발견이면 → "증빙서류에서 집행금액 미확인" 플래그

## 증빙 충족 판정 원칙
1. **PDF 증빙이 있고 금액 확인** → 기본적으로 적정
2. **PDF 증빙이 있으나 금액 미확인** → 확인 (이미지 PDF인 경우 SKIP)
3. **증빙 미첨부** → 확인 (증빙서류 없음)
4. **비목별 한도 초과** → 확인 (criteria에 명시된 한도 기준)

## 이미지 PDF 판별
- PDF 텍스트 추출 결과 한글/숫자 비율 30% 미만이면 이미지(스캔) PDF로 판정
- 이미지 PDF는 금액 대조 SKIP (OCR 정확도 보장 불가)

## 0원 항목 처리
- 금액이 0원인 항목은 취소전표로 간주하여 SKIP 처리
