## 1) 작업순서 고정

1. 기준 시트: `상시`
2. 처리 순서: 시트의 행 순서(작업순위) 그대로
3. 이전 건 완료 후 다음 행으로 이동
4. 완료 기준: 해당 기관에서 `미확정 0` AND `검토중 0`

---

## 2) 진입 규칙 고정

1. `상시점검 보고서 생성대상 여부` 체크 해제
2. 검색은 `과제번호`만 입력
3. 결과 목록에서 `연구수행기관` 문자열 대조 후 진입
4. `사업연도`는 필터에 입력하지 않고 대조만 수행
5. 과제번호가 동일한 다기관 과제에서는 기관 대조 없이 진입 금지

---

## 3) 판정 방법 (항상 동일)

1. 기준서 우선순위:
   - `projects/이지바로-공통/criteria-v3.md`
   - 본 문서(`운영-기준-v2.md`)
2. 건별 판정 순서:
   - 비/세목 확인
   - 집행방법 확인(`현물` 여부)
   - 금액 필드 확인(공급금액/연구비이체금액)
   - 증빙 팝업 열기
   - 첨부파일명 확인(필요 시 본문 확인)
   - 기준서 매칭 후 `정상/미흡/미확정유지` 결정
3. 판정 원칙:
   - 명확 근거가 있는 경우만 `정상`
   - 명확 위반만 `미흡`
   - 애매하면 `미확정 유지` (정상 일괄 금지)

---

## 4) 세부 판정 규칙(핵심)

1. 연구재료비:
   - 기본 3종(내부결재/거래증빙/검수확인) 확인
   - 3종 충족 + 연관성 확인 시 `정상`
   - 누락 명확 시 `미흡`
2. 영리기관 부가세:
   - 영리기관이고 `연구비이체금액 > 공급금액`이면 `미흡`
   - 점검의견: `관세 및 부가세 미환급`
3. 집행방법 `현물`:
   - 증빙 부재만으로 `미흡` 금지
   - `정상` 처리 가능(로그에 현물 적용 표시)
4. 증빙 확인:
   - 파일명만으로 확정 금지
   - 필요 시 본문 확인 후 최종 판정

---

## 5) 상태 반영 규칙

1. 판정 적용 후 `저장`
2. 팝업 발생 시 종류와 무관하게 모두 `확인` 클릭
   - 예: `저장되었습니다`, `알림`, `오류가 발생했습니다`
3. `검토중` 필터 조회
4. 대상 전건 선택 후 `확인완료`
5. 후속 팝업 전부 `확인`
6. 최종 수치 재확인: `미확정 0`, `검토중 0`

---

## 6) 금지사항

1. 근거 없는 `정상` 일괄처리 금지
2. 과제번호/기관 하드코딩 금지
3. 팝업 미확인 상태로 다음 건 이동 금지
4. 로그 미기록 상태에서 완료 처리 금지

---

## 7) 로그 필수

1. 기관 단위로 JSON + XLSX 저장
2. 필수 컬럼:
   - 순번, 결의순번, 결의제목, 비/세목
   - 처리결론, 처리사유, 주요 근거(파일명/핵심필드)
   - 처리 전/후 요약(미확정/검토중/정상/미흡/점검완료)
3. `projects/progress.md`에 즉시 반영
